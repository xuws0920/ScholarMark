# ScholarMark

一个基于浏览器的 PDF 文献阅读、标注与笔记工具，支持 Markdown 与公式渲染、图表摘录、文献总结卡片库，以及本地自动保存。

## 本地环境配置

### 1. 环境要求

- Node.js `>= 18`（建议 `20+`）
- npm `>= 9`
- 现代浏览器（推荐 Chrome / Edge）
- Windows / macOS / Linux

### 2. 安装依赖

```bash
npm install
```

### 3. 启动开发环境

```bash
npm run dev
```

默认地址：

- `http://localhost:5173`

### 4. 构建生产版本

```bash
npm run build
```

### 5. 预览生产包

```bash
npm run preview
```

### 6. 编码检查（防乱码）

```bash
npm run check:encoding
```

并且 `npm run build` 前会自动执行编码检查（`prebuild`）。

## 本地自动保存配置

项目支持将笔记/总结自动保存到本地目录（File System Access API）。

### 首次配置

1. 打开应用后点击右上角设置
2. 选择存储目录（例如：`D:\Work\note\文献笔记`）
3. 授权写入权限

### 保存行为

- 笔记编辑后自动保存（防抖）
- 文献总结编辑后自动保存（防抖）
- 保存结构：
  - `文献名/笔记名.md`
  - `文献名/文献总结.md`

### 注意事项

- 首次必须手动选择目录，浏览器不允许静默指定绝对路径并授权
- 清理站点权限/站点数据后，需重新选择目录

## PM2 本机常驻（固定 localhost）

适合持续开发，免去每次手动启动命令。

### 1. 全局安装（一次）

```bash
npm i -g pm2 pm2-windows-startup
```

### 2. 启动服务（使用固化参数）

```bash
pm2 start ecosystem.config.cjs
```

### 3. 设置开机自启（一次）

```bash
pm2-startup install
pm2 save
```

### 4. 常用管理命令

```bash
pm2 status
pm2 logs pdf-reader
pm2 restart pdf-reader
pm2 stop pdf-reader
pm2 delete pdf-reader
pm2 restart ecosystem.config.cjs
```

### 5. 更新代码后的建议流程

如果只改了源码（`src/`、`index.html`、样式）：

```bash
pm2 restart pdf-reader
```

只有在依赖变化时（`package.json` / `package-lock.json` 变更）才需要：

```bash
npm install
pm2 restart pdf-reader
```

### 6. 访问地址

- `http://localhost:5173`

## 功能介绍

### 1. 文献库与布局

- 导入 PDF（文件选择 / 拖拽）
- 左侧栏支持 `文献库 / 卡片库` 两入口
- 左侧栏可收纳，右侧栏可大范围拉伸
- 左侧栏收纳后提供快捷图标入口：
  - `文献库`：打开上层抽屉查看已导入文献
  - `卡片库`：作为快捷入口直达原有全屏卡片库（逻辑不变）
- 删除文献时级联删除关联数据（标注、笔记、总结、图表摘录、总结卡片）
- 支持重命名已导入文献（自动去除 `.pdf` 后缀，并同步到中栏标题、导出目录与卡片显示）
- 文献列表每项右侧提供外置竖向操作图标（重命名 / 删除）

### 2. PDF 阅读

- 翻页、缩放、适应宽度
- 文本层可选中
- 阅读进度记忆（按文献保存页码与缩放）
- 顶栏文献名支持悬浮显示完整文献全称（Tooltip）
- 工具栏支持打开“文献目录”抽屉（中栏左上角入口）
- 工具栏支持 PDF 内查找（按钮触发悬浮面板）
- 工具栏支持“评论管理”侧边面板入口
- 查找面板支持：
  - 上一项 / 下一项
  - 命中计数
  - `Enter / Shift+Enter` 快捷跳转
  - `Esc` 关闭/清空
  - 严格大小写匹配（`Aa` 开关）
  - 面板拖动（在 PDF 区域内移动）
- 支持“论文 + 全文翻译预览”双页并排模式（中栏按钮切换）
- 双页模式下左右区域滚动独立，鼠标滚轮只作用于悬停区域

### 3. 标注系统（增强）

- 选中文本后支持三类标注：高亮 / 下划线 / 删除线
- 标注样式按钮采用图标化操作（线性图标）
- 支持颜色抽屉选择，颜色作用于当前标注样式
- 标注项支持编辑：
  - 标注片段修正（Markdown/公式）
  - 对应问题（Markdown/公式）
- 标注列表直接渲染 Markdown 与公式
- 删除标注支持级联删除笔记中关联整块内容（含引用块和块内编辑内容）
- 删除前提供内容预览确认
- 新建标注区分入口来源：
  - `插入到笔记` 创建：在“笔记 Tab > 标注”与“评论管理”均可见
  - `高亮/下划线/删除线` 创建：仅在“评论管理”中展示
  - 历史数据按旧规则保留（仅新建数据按新规则生效）

### 4. 笔记系统（Markdown）

- 多笔记管理（每篇 PDF 独立）
- 顶部 Tab 排布：目录 / 编辑 / 预览 / 标注
- 自动保存到本地数据库
- 公式渲染（KaTeX）
- 代码高亮（highlight.js）

### 5. 笔记内目录（基于标注引用）

- 目录以浮层方式显示在中栏上层（不占用右栏编辑区）
- 目录项来自标注引用句子，不依赖 Markdown 标题
- 点击目录项可跳转到编辑/预览对应引用块
- 长句自动截断，悬浮显示完整内容
- 浮层位置与高度随窗口和布局自适应更新

### 6. 文献总结模块

- 独立于笔记的总结工作区（编辑 / 预览）
- 默认模板：研究内容、研究方法、研究结果、讨论
- 支持重置模板与导出
- 支持“导出为卡片”（按文献覆盖更新，并生成卡片缩略图）

### 7. 概览工作区

- 工作区新增 `概览`
- `总览` 子页：
  - 标注数、问题数、笔记数、图表摘录数
  - 当前阅读进度与最近阅读时间
  - 热点页、最近问题（可点击跳转）
- `图表摘录` 子页：
  - 框选 PDF 区域生成图表摘录
  - 卡片网格浏览
  - 点击卡片后打开编辑抽屉，编辑标题/标签/说明
  - 支持跳转原页、删除、保存

### 8. 卡片库（全屏）

- 左侧入口 `卡片库` 进入全屏浏览
- 左栏收纳态下可通过快捷图标入口直接打开全屏卡片库
- 每篇文献最多一张总结卡片（由“导出为卡片”生成）
- 左侧目录按当前排序展示卡片条目（文献名 + 更新时间 + 缩略图），支持点击跳转
- 支持左右切换上一张/下一张
- 显示卡片总数与当前序号
- 支持排序切换：按更新时间 / 按文献名
- 历史卡片若无缩略图，可通过重新“导出为卡片”补齐

### 9. 搜索与联动

- 全局搜索标注与笔记内容
- 点击结果联动到对应文献与位置

### 10. 导出

- 导出当前笔记
- 批量导出全部笔记
- 导出文献总结
- 导出当前翻译与全文翻译
- 手动导出输出 `MD`
- 支持浏览器下载与目录写入

### 11. 翻译工作区（截图驱动）

- 顶层工作区新增 `翻译`
- 子页：
  - `翻译工作区`：框选翻译、整页翻译、复制、重译、范围翻译
  - `全文翻译`：目录 / 编辑 / 预览、目录导航、导出
- 默认输出中文译文（术语优先，公式尽量保留）
- 支持 OpenAI-compatible 接口配置（Base URL / API Key / Model / 字段映射）
- 全文翻译目录在编辑 Tab 下支持稳定跳转（编辑态与预览态均可定位）

### 12. 目录抽屉统一化

- 文献目录：中栏左上角按钮打开抽屉
- 笔记目录：笔记 Tab 区按钮打开浮层目录
- 全文翻译目录：全文翻译子 Tab 区按钮打开浮层目录
- 三类目录均支持：
  - 打开/关闭过渡动画
  - 基于视口与布局的自适应高度
  - 内容较多时内部滚动

### 13. 收纳态文献抽屉

- 入口位于左栏收纳态快捷图标区（文献库图标）
- 以抽屉形式显示在左侧内容区上层
- 支持按文献名搜索、点击文献直接打开
- 抽屉高度按内容与可用空间动态适配：
  - 内容少时自动收缩
  - 内容多时抽屉内部列表滚动

### 14. 评论管理（右侧面板）

- 可按页查看当前文献全部标注（含高亮/下划线/删除线）
- 点击 PDF 原文标注可联动定位到对应评论项
- 点击评论项可反向跳转到 PDF 原文位置
- 每条评论支持直接删除（无二次确认），删除后同步移除原文标注
- 评论输入框默认单行显示，输入内容时自动增高
- 评论内容自动保存到本地数据库（IndexedDB）
- “导出带批注 PDF”按钮当前为占位提示，尚未启用 PDF 写回

## 常见问题

### 1. 为什么每次都要重新选存储目录？

通常是因为浏览器站点权限或站点数据被清理。重新选择目录并授权即可恢复。

### 2. 数据会不会丢？

本地数据依赖浏览器存储。清理站点数据、无痕模式结束、重装系统/换设备都可能导致数据不可恢复，建议定期导出备份。

### 3. 为什么 `localhost` 和 `127.0.0.1` 数据看起来不一致？

两者属于不同站点 origin，本地存储彼此隔离。请固定使用同一地址（推荐 `http://localhost:5173`）。

### 4. 为什么 LaTeX 的 `table/tabular` 在预览里不能直接渲染？

当前预览使用 Markdown + KaTeX，主要支持 `$...$` 与 `$$...$$` 公式定界。`\\begin{table}` / `\\begin{tabular}` 等 LaTeX 表格环境不在默认支持范围内，建议改用 Markdown 表格并在单元格中嵌入公式。
